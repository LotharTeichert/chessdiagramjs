Living Chess Diagrams

Reading articles on Chess is troublesome to me, having to concentrate hard to visualize the situation on the board after some moves. On computers, you have easy support by lots of software solutions to display games and problems in prepared form. Searching on the internet I found Chessboardjs by Chris Oakman, a javascript that makes best animation of chess moves in my opinion. For simple integration in web sites with commenting text I created the javascript ChessDiagram.

In the menu you will find some pages demonstrating the virtue of these diagrams. For two of them (Lucena and Morphy) I simply copied the text of a wikipedia page and replaced the picture diagrams. The commented moves now can be played in the diagrams and you can try your own moves as well. (Note that pawn promotion will always give a queen at first, but you can drag the promoted piece off the board to get another one.)

Usage

To use the diagrams on HTML page, you have to link a style sheet and four scripts in the header part: 
<html>
<head>
 <title>Lucena Position
 <link rel="stylesheet" href="../css/chessboard-0.3.0.css" />
 <script type="text/javascript" src="../js/json3.js">
 <script type="text/javascript" src="../js/jquery-2.0.3.min.js">
 <script type="text/javascript" src="../js/chessboard-0.3.0.js">
 <script type="text/javascript" src="../js/chessdiagram.js">
</head>
...
Now the body may contain sections like: 
...
<div id="diag01"></div><div id="diag01_pgn"></div>
<script>
ChessDiagram.diagram('diag01', 
'rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1',
'1. e2-e4 e7-e5 2. Sg1-f3 d7-d6 3. d2-d4 Lc8-g4?! 4. d4xe5 Lg4xf3 ' +
' 5. Dd1xf3 d6xe5 6. Lf1-c4 Sg8-f6 7. Df3-b3 Dd8-e7 ' +
'8. Sb1-c3 c7-c6 9. Lc1-g5  b7-b5? ',
'Morphy – Karl von Braunschweig und Graf Isouard, Paris 1858<br />',
'',
{ divStyle: { backgroundColor:'#e0e0e0', 
              width:'290px', 
              styleFloat:'left', cssFloat:'left', 
              margin:'8px', 
              padding:'4px' }} );
</script>
...
It's not necessary to follow the div-tag immediately by the script. You may put all scripts in a collection at the end of the page.

The parameters of ChessDiagram.diagram are:

  • The Id of the div-tag to display the board. Another Id is derived from that one by appending '_pgn'. It determines the div-tag containing the moves and comments. If this div-tag is not found on the page, the diagram will contain another button [+] to display this text below the diagram when clicked.
  • A description of the position as a FEN string (see wikipedia for explanation).
  • The move list to be shown. Permitted is algebraic notation, long or short, including variations and comments, move numbers and dots allowed. That should make it as simple as possible to generate a move list from existing text. The language in which moves are given, as well as the language in which the moves are displayed, can be configured.
  • A header for the diagram, possibly HTML formatted.
  • A footer for the diagram, possibly HTML formatted.
  • Additional options (in curly brackets). Might be missing (one parameter less) and will hold until the next call to ChessDiagram.diagram with options present. Some of them are only for specialists. They are:
    • divStyle: additional style attributes for the div-tag. Handy to format all diagrams in one place (the first call to ChessDiagram.diagram).
    • showNotation: 1 (default) for traditional board notation, 2 for ChessBoard board notation (inside board), 3 for no board notation
    • boardWidth: width of the board (the 8 x 8 squares), default: '240px'
    • pieceTheme: option from ChessBoard, default: chesspieces/wikipedia/{piece}.png in the graphic directory
    • imgPath: path of the graphic files. Not necessary if the graphic directory is a sibling of the script directory and it's name is img.
    • notationStyle: style (like a HTML style string) for a table containing the board. The first and last row and column of this table contain the traditional board notation (if selected), so you can format it with this string. Default: 'background-color:#f0f0f0;font-family:Lucida Console;font-size:12px;line-height:200%'
    • orientation: 'white' (default) or 'black', decides the side to look at the board.
    • pgnFormat: 1 (default) makes a simple formatting of moves and comments: moves of the main line are bold, moves of variations italique; a change to comments or variations within the main line is emphasized by a linebreak. pgnFormat = 0 turns this off and you have to make HTML-formatting in the text by yourself.
    • languageIn: The language (in English), used by the the move list, default is 'English'
    • languageOut: The language (in English), used when the moves are diplayed in the page, default is 'English', 'Unicode' is possible.
